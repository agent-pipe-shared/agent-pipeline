# ADR-0018: Retro-Prozess-Revision — Elephant verfasst das Close-Retro selbst

> _A German version follows below · Eine deutsche Fassung folgt weiter unten._

**In brief (English):** This ADR retires the ritual close-of-session retro question ("what should the pipeline do better next time?") that used to be posed to the PO. Instead, the session Elephant now writes the close-retro itself — either concrete improvement items or an explicit "nothing" — as a backlog item/handoff to the pipeline Elephant's continuous-improvement process, and it must never be silently skipped. The PO may still contribute observations separately, through their own channel, without a ritual prompt. Status: accepted 2026-07-04.

> Agent-Pipeline v0.1.0-draft · Sprint 0 Phase 4 · Stand 2026-07-06

**Status:** akzeptiert (2026-07-04, <PROJECT_B>-S38-Close) · **Grundlage:** Register E18

## Kontext

Die bisherige Praxis stellte die Retro-Pflichtfrage („Was soll die Pipeline nächstes Mal besser machen?") ritualisiert an the PO. the PO direktierte am <PROJECT_B>-S38-Close, dass der Session-Elephant das Close-Retro künftig selbst verfasst, statt es abzufragen.

## Entscheidung (E18, wortgetreu)

> Retro-Prozess-Revision (the PO-Direktive, 2026-07-04, <PROJECT_B>-S38-Close): Die Retro-Pflichtfrage („Was soll die Pipeline nächstes Mal besser machen?") wird NICHT mehr an the PO gestellt — der Session-Elephant verfasst das Close-Retro SELBST (konkrete Items oder explizites „nichts") als Backlog-Item/Transfer an den Pipeline-Elephant (KVP); nie still ausgelassen. the PO reicht eigene Beobachtungen nebenbei ein (eigener Kanal, keine Ritual-Abfrage). Umsetzung: close-block Schritt 7 + operating-model §7 + Kickoff-/Bootstrap-Verweise (AP-Sprint Paket 3, 2026-07-04).

## Konsequenzen

**Positiv:** Entlastet the PO von einer wiederkehrenden Ritual-Frage; das KVP läuft dennoch verlässlich weiter, weil „nie still ausgelassen" gilt.

**Negativ:** Selbstreflexion durch dieselbe Instanz, die die Session geführt hat — Risiko von Betriebsblindheit.

**Risiko:** Der Elephant könnte das Retro pro forma abhaken statt substanziell. Mitigation: ein explizites „nichts" ist zulässig, muss aber ausgesprochen sein — kein stilles Weglassen.

## Verworfene Alternativen

- **Beibehaltung der Ritual-Abfrage an the PO** — genau das, was die Revision abschafft, weil sie wiederkehrenden Aufwand ohne belegten Mehrwert erzeugte.
